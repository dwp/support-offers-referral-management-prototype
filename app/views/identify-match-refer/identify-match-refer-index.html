Duplicate referrals
Link to duplicate referrals prototype
Summary
For iteration 1, we will only be able to identify duplicates at the end of the journey. Instead of a confirmation page, the user will see an error screen. The system will only identify duplicates from any referrals that have gone through our system previously. The user should still be checking LMS and F&R for duplicates across other systems.

Version 1.0 duplicate referrals tested

Supporting information

Link to prototype
Version 1.0 duplicate referrals prototype
Sprint version
Sprint 0 Beta
Summary
If a citizen has already been referred to a national contracted provision they are not eligible to be referred to any other national contracted provision unless they either complete the programme or, the provider rejects them or cancels the referral. The technical solution can identify the person and their referral history at the starting point of the service and the UI prevents the work coach from trying to refer the citizen unnecessarily.
